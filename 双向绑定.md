

### 1. 单向数据流（One-way Data Flow）

**定义：**
- 数据只从一个方向流动，通常是从父组件 → 子组件。
- 状态（state）变化只会影响下游（子组件），子组件无法直接修改父组件的数据。

**优点：**
- 数据流向清晰，易于调试和维护。
- 适合大型应用，状态管理更可控。
- 易于追踪数据变化，利于性能优化。

**缺点：**
- 某些场景下，数据传递需要层层 props，操作较繁琐。

**主流框架：**
- **React** 采用单向数据流，父组件通过 props 传递数据给子组件，子组件通过事件回调通知父组件更新。

---

### 2. 双向数据绑定（Two-way Data Binding）

**定义：**
- 数据和视图互相绑定，任何一方变化都会自动同步到另一方。
- 典型场景：表单输入，数据变化自动更新视图，视图变化也能自动更新数据。

**优点：**
- 编码简洁，表单等场景开发效率高。
- 适合中小型应用，交互频繁时体验好。

**缺点：**
- 数据流动方向复杂，调试困难，状态难以追踪。
- 大型项目中可能导致数据混乱，性能难以优化。

**主流框架：**
- **Vue** 提供双向绑定（如 `v-model`），底层通过 MVVM（Model-View-ViewModel）模式实现。
- Angular 也支持双向绑定。

---

### 3. 框架举例

- **React**：单向数据流，数据通过 props 传递，事件回调（如 onChange）通知父组件更新状态。
- **Vue**：支持双向绑定（如 `v-model`），也支持单向数据流（如 props），开发者可根据场景选择。

**代码示例：**

**React 单向数据流**
```jsx
function Parent() {
  const [value, setValue] = useState('');
  return <Child value={value} onChange={setValue} />;
}

function Child({ value, onChange }) {
  return <input value={value} onChange={e => onChange(e.target.value)} />;
}
```

**Vue 双向绑定**
```vue
<template>
  <input v-model="value" />
</template>
<script>
export default {
  data() {
    return { value: '' }
  }
}
</script>
```

---

**总结：**
- 单向数据流：数据流动方向明确，适合大型项目（React）。
- 双向数据绑定：开发效率高，适合表单等场景（Vue、Angular）。
